HTML

  • The head element contains information, or metadata, about the webpage that the user almost never see
  • Metadata is used by the browser to render things. It is also helps with performance and search engine recognition
  • The title element is used for the title of the webpage and is very crutial for SEO (Search engine optimization)
  • The body element represents the visible content shown to the user
  • All elements we want the user to interact with have to go in the body element
  • !DOCTYPE tells the browser what language the code is written in and it must be declared at the top
  • HTML attributes define more info about the element, example: "lang="en" would lets us know the page is in English
  • Elements nested inside of other elements are known as child elements.
  • Semantic HTML elements are elements that use words to clearly represent what the element contains in human language. It is best practice to use these when able because it makes the website accessible to more users like people who used screen readers for example
  • Non-semantic elements are used for structural issues that developers wants to solve such as div to create a division in the page
  • Make sure you close your tags off. Example div /div

CSS

  • CSS is used by devs that define how website content is presented ti users (colors, aesthetics, visual layouts, etc).
  • Inline CSS tags are used to style specific HTML elements and they are really only used for more permanent changes, but they are less flexable than internal or external style sheets.
  • Internal style sheets contain rules in the head section of the HTML file and those rules only apply to that page. Example: using " style h 1 color: red" will apply the color red to all h1 headings in that HTML file.
  • External style sheets are the most common way to style a web page because is seperates all CSS code that can be link to the HTML file with the simple link tag (example doesn't show full function)
  • Margins in CSS indicates how space we want around the element itself, but padding indicates how spacing we want inside the element from the edge.
  • When we want all visual elements on the page to have a certain style we use the "*" at the beginning out our tag
  • You can also style multiple HTML elements by seperating with a common in our CSS file (example: header, footer {})
  • Do Not Repeat Yourself is a coding principle help reduce the overall size of the source code and complexity
  • Assigning colors can be done by using English like gray or can be done with numbers like 88888 to get a certain shade

Git

  • git status: checks what branch we are currently working in
  • git checkout -b branch name: creates a new branch to work in our repo
  • git checkout branch name: allows us to switch what branch we are in
  • cd file: lets us choose what directory we are working in for our local PC
  • git clone SSH key: allows us to clone our repo in out local environment to begin work
  • "touch" allows you to create a new file in directory via gitbash

      • Git Flow

    1. git add -A
    2. git commit -m "message"
    3. git pull origin "main branch"
    4. git push origin "the branch you are working in"
    5. git checkout "main branch"
    6. git pull origin "main branch"

JavaScript

  • Semi colons are only needed if you are trying to seperate statements on a single line, howevere it is best practice to use them as often as possible
  • You can add "//" to comment on a certain line of code in JS or /* message */ as this tells the browser to ignore the message
  • a variable is a container to store data inside our code "var x = 0;"
  • strings have to be in quotations marks
  • Control flow is the order in which the computer executes code, from top to bottom
  • Conditional logic in JS is used to control the order in which code is executed, or if it is all. If the condition is falsy(0, -0, On, "", null, undefined, Nan) then that statement will be ignored and the computer will try to find a truthy statement and run that.
  • Running "if" statements in JS introduces decision making in our code. We can list multiple conditions in our "if" statement and the computer will go through each condition until it finds one that is truthy and it will proceed with that is between the curly brackets. Use "else if" for each condition and "else" if nothing is defined.
  • An array is a single variable that is used to hold a group of data. Declaring an array is the same, but you just use "[x,x,x,x]" instead of the singular variable.
  • Array's can hold more than one type of data like strings, numbers, and Boolean value. Strings have to have quatations mark around them, numbers and Boolean values don't. "var shapes = ["triangle", "square", "pentagon", "circle"];" The shapes listed would be considered a string (string of words)
  • When an array has multiple data entries, they are each assigned an index according to the order in which it is stored in the array and the pattern is 0, 1, 2, 3...
  • console is a test environment that devs use to test code, the .log() will output whatever is inside is inside the ()
  • when linking you JS script in HTML, it is best to link right before the body close ( /body )
  • for loop: "for(var x = 0; x < shapes.length; x++) console.log(shapes[x]; ". Here our for loop is saying that x is equal to 0. When x is less than the length of the shapes array, add 1 to x. That way we get our loop to log each item in the array in one call.
  • functions need to be first defined and then called later on. I can write "function HelloWorld () { console.log('Hello Everyone!');}. Nothing will happen until I call that function later on. Example: HelloWorld();. "
